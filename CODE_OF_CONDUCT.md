# Code Of Conduct

## Expected Behavior

Participants are expected to keep discussions respectful, technical, and inclusive. Good-faith disagreement is welcome when it is specific, evidence-based, and focused on improving the project.

Expected behavior includes:

- using welcoming and professional language
- respecting differing experience levels and viewpoints
- giving actionable feedback instead of personal criticism
- accepting maintainer decisions once a direction is resolved
- reporting safety, abuse, or conduct concerns privately

## Unacceptable Behavior

Unacceptable behavior includes harassment, discriminatory language, threats, doxxing, sustained disruption, unwanted sexual attention, or publishing private information without permission.

Security reports must not be used to expose secrets, exploit details, or sensitive operational evidence publicly.

## Enforcement

Maintainers may edit or remove comments, close issues or pull requests, temporarily restrict participation, or permanently block contributors when behavior creates risk for the project or community.

Report conduct concerns to the project maintainers through a private channel. Maintainers should acknowledge reports, review available context, and take proportionate action while protecting reporter privacy where practical.
